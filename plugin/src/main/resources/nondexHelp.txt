A tool for finding strong assumptions on APIs with non-deterministic specifications.
See https://github.com/TestingResearchIllinois/NonDex

This plugin has 3 tasks:

nondexTest
    Run tests with nondex and find out the non-deterministic issue
    Use the -DnondexRuns command line argument to run tests a specified number of times
    Use the -DnonexSeed command line argument to run tests with a specific seed of randomization
    Filtering to only run certain test methods or classes can be done in same way as default Test task

nondexClean
    Remove the .nondex directory that is created when tests are run with nondex

nondexHelp
    Print this help message
